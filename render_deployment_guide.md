# 🚀 ATSify Render Deployment Guide

This guide details the step-by-step process of hosting the entire ATSify application (Frontend, Backend, and AI Service) on **Render** (Free Tier) with a remote **MongoDB Atlas** database.

---

## 🏗️ Deployment Architecture

We use a consolidated **2-Service Architecture** to maximize Render's Free Tier resources and eliminate CORS complexities:
1. **Service 1: AI NLP Service (Python FastAPI)**
   - Deployed as a Python Web Service on Render.
2. **Service 2: Web Server (Node.js/Express + React Client)**
   - Deployed as a Node Web Service. Express is configured to build and serve the React build folder (`frontend/dist`) statically.
3. **Database: MongoDB Atlas**
   - Free-tier MongoDB cluster on cloud.

---

## 💾 Step 1: Set Up Free MongoDB Atlas Database

Since Render does not offer a free native MongoDB database, you must create a free tier cluster on MongoDB Atlas:

1. Sign up/Log in to [MongoDB Atlas](https://www.mongodb.com/cloud/atlas).
2. Create a new project (e.g., `ATSify`).
3. Click **Build a Database** and select the **M0 Free Tier** cluster.
4. Set up database security access:
   - **Username/Password**: Create a user and save the password securely.
   - **IP Access List**: Set it to **Allow Access from Anywhere** (`0.0.0.0/0`) so Render services can connect dynamically.
5. Once the cluster is created, click **Connect** $\rightarrow$ **Drivers** $\rightarrow$ **Node.js**.
6. Copy the connection string. It will look like this:
   `mongodb+srv://<username>:<password>@cluster0.xxxxxx.mongodb.net/?retryWrites=true&w=majority&appName=Cluster0`
7. Replace `<password>` with your database user's password and save this URL for later.

---

## 🐍 Step 2: Deploy AI Service (FastAPI) on Render

1. Log in to your [Render Dashboard](https://dashboard.render.com).
2. Click **New +** $\rightarrow$ **Web Service**.
3. Connect your GitHub repository: `gova-tech-25/ai-resume-analyser-ats`.
4. Configure the service with the following settings:
   - **Name**: `atsify-ai-service`
   - **Root Directory**: `ai_service`
   - **Runtime**: `Python 3`
   - **Build Command**: 
     ```bash
     pip install -r requirements.txt && python -m spacy download en_core_web_sm
     ```
   - **Start Command**: 
     ```bash
     uvicorn app.main:app --host 0.0.0.0 --port $PORT
     ```
   - **Instance Type**: Select **Free** tier.
5. Click **Create Web Service**.
6. Once deployed, copy the **AI Service URL** generated by Render (e.g., `https://atsify-ai-service.onrender.com`).

---

## 📦 Step 3: Deploy Backend & Frontend (Unified Express App) on Render

1. On your Render Dashboard, click **New +** $\rightarrow$ **Web Service**.
2. Connect the same GitHub repository: `gova-tech-25/ai-resume-analyser-ats`.
3. Configure the service with the following settings:
   - **Name**: `atsify-web-server`
   - **Root Directory**: `backend`
   - **Runtime**: `Node`
   - **Build Command**: 
     ```bash
     npm install && npm run build
     ```
     *(Note: This installs backend packages and triggers our package.json script to install and build the React frontend automatically!)*
   - **Start Command**: 
     ```bash
     npm start
     ```
   - **Instance Type**: Select **Free** tier.
4. Open the **Environment** section (under the settings page) and add the following Environment Variables:

| Key | Value | Description |
| :--- | :--- | :--- |
| `NODE_ENV` | `production` | Enables production Express static file serving. |
| `MONGODB_URI` | `[Your MongoDB Atlas connection string]` | Saved from Step 1. |
| `AI_SERVICE_URL` | `[Your Render FastAPI service URL]` | Saved from Step 2 (e.g., `https://atsify-ai-service.onrender.com`). |

5. Click **Create Web Service**.
6. Once deployed, click the **Web Server URL** (e.g., `https://atsify-web-server.onrender.com`) to open your live application!

---

## ⚙️ How the Unified Build Works

Under this setup, when Render triggers a deployment:
1. It downloads your project and navigates to the `backend/` folder.
2. It runs `npm run build` inside backend, which executes:
   `npm install --prefix ../frontend && npm run build --prefix ../frontend`
3. This command compiles the React project and places the build folder at `frontend/dist`.
4. The Express backend serves those built static assets. Since both backend APIs and React static files run on the same port, the browser does not trigger CORS warnings, and you only require one Web Service slot!
